# Task: Discussion

> Discussion 要回答的问题：**这些 results 意味着什么 + 我们的工作放在 broader context 里有什么 implication**。
> 服务的 reviewer 决策：研究有没有 broader impact + 解读是否合理 + 局限性是否诚实。

## ⚓ 元层锚点

- `meta/constitution.md`：现在时（解读）+ 过去时（重述发现）+ 现在时 + 情态动词（推测）
- `meta/over_claim_guard.md`：**Discussion 是 over-claim 高发区**，每个 implication 都要审
- `meta/reviewer_psychology.md`：reviewer 在这里找 "so what"
- `meta/workflow.md`：Frame 时明确 main message

---

## 任务本质

Discussion 不是"重写一遍 Results"。Discussion 是 **把 results 放到 broader scientific context 里**——和前人对话、解读机制、提出 implication、诚实承认局限。

Discussion 写得好的标志：reviewer 看完后能在脑里**给你的研究在该领域知识图谱里找到位置**。

---

## 通用工作流

### Frame
- 我的 1-2 个**核心 take-home message** 是什么？(超过 2 个 = 主线散)
- 这些 take-home 在该领域**填补了什么空白**？(回到引言 §3 的 gap)
- 我有什么**核心 limitations** 必须诚实写？

### Outline
默认 4 段（数量看篇幅可缩到 3 或扩到 6，但**不要超过 6**）：

```
§1 [核心发现重述 + 一句话 take-home]
   - 1 句话：我们做了什么 + 找到了什么
   - 1-2 句话：核心 take-home

§2-3 [机制解读 / 与前人对话]
   - 把发现 X 放到已有文献里
   - 一致 / 不一致 / 扩展前人
   - 引用密度高（每段 3-7 个引用）

§4 [Implication / Broader context]
   - 这个发现对该领域意味着什么
   - 对应用 / 育种 / 临床 / 保育的潜在价值
   - 用 may / could / suggest（不要 will）

§5 [Limitations]
   - 主动列 2-4 条
   - 不要装鸵鸟也不要自我否定
   - 每条 limitation 给"未来可以怎么做"

§6 [Conclusion 或 Outlook]（可选）
   - 1 段，3-5 句
   - 强 take-home + 未来方向
```

### Draft
每段开头是 1 句明确的 statement——不是含糊的过渡。

### Self-review
- 每个 implication 是不是 over-claim？(参考 over_claim_guard.md)
- 与前人对话时有没有贬低？
- limitations 是不是真的 limitations？(还是装样子)

---

## 通用骨架（4 段最简版）

§1 — Take-home：
- 第 1 句：本研究做了什么 + 主要发现
- 第 2-3 句：1-2 个核心 take-home（用现在时 + 适度强表达）

§2 — 解读：
- 把核心发现放到机制 / 已知功能里
- 与前人对话（引用 3-5 个原创研究）
- 强调你的研究**新加的洞察**

§3 — Implication：
- 对该领域 / 应用的潜在 implication
- 用情态动词（may / could / suggest）

§4 — Limitations + Future：
- 2-4 条诚实的 limitation
- 每条配"未来怎么做"

---

## 核心结构原则

### "倒沙漏" 而非"漏斗"
- Introduction 是漏斗（broad → specific）
- Discussion 是**倒沙漏**（specific findings → broader context）
- §1 specific findings → §N broader implication

### 与 Introduction 闭环
- Introduction §3 提了 gap A, B, C
- Discussion 必须回应 A, B, C：填了多少 / 还剩什么
- Introduction §4 承诺做了 X, Y, Z
- Discussion 必须解读 X, Y, Z 的意义

### 不要重复 Results
- Results 已经说"我们发现 184 个候选基因 (P < ...)"
- Discussion 说"这 184 个基因中，*TSHR* 和 *LCT* 已知与 X 相关..."（**进一步解读**）
- 不要"我们发现 184 个候选基因 (P < ...)"再重复一遍

## 元层雷区（跨方向通用）

### 1. 全篇过度推测
- 用 "may" / "could" / "suggest" 是好的
- 但每段都"may suggest"+"may indicate"+"could imply" = 没立场
- **强证据时该强表达就强**（参考 over_claim_guard.md 反向校准）

### 2. 没立场
❌ "Some studies suggest A. Other studies suggest B. Both have merit."  
✅ "Our results support A, but the strength of evidence is sensitive to [specific factor]."

reviewer 期待你**有立场**——基于你的数据告诉他你倾向哪一边。

### 3. Implication over-claim
❌ "This will revolutionize cattle breeding."  
✅ "These candidate genes provide targets for marker-assisted selection in [specific breeding context]."

参考 `over_claim_guard.md`#应用前景。

### 4. 假性 limitations
❌ "A limitation is that we did not include more samples."（每篇都能这么说）  
✅ "A limitation is that our 5 northern breeds could not fully capture the climatic gradient between 60°N and 70°N; future sampling in [specific region] would test whether [specific prediction] holds."

具体的 limitation + 具体的 future direction。

### 5. 不诚实的"local optimum"
❌ "Our method outperforms all previous methods."  
✅ "Our method shows improved performance over [specific methods] in [specific aspect], although [other methods] may be more suitable for [other aspect]."

诚实比 over-claim 安全——reviewer 自己也是该领域人，会知道你 over-state。

### 6. Discussion 篇幅失控
- 顶刊 Discussion 通常 1500-3000 词
- 二区可以更长
- **失控信号**：6 段以上 / 每段都讲一个候选基因 / 重复 results 的数字 → 砍掉 30%

### 7. 引用偏向
- Discussion 引用密度比 Introduction 还高（每段 3-7 个）
- 偏向自引 = reviewer 警觉
- 偏向单一团队 = reviewer 怀疑
- 平衡引用：本领域 + 跨领域 + 经典 + 近期

### 8. 与前人贬低式对比
❌ "Previous studies failed to identify ..."  
✅ "Previous studies, limited by [specific constraint], could not address ..."

参考 `over_claim_guard.md`#比较替代。

### 9. Conclusion 写成 Abstract
- Conclusion 段是 Discussion 末段，**不是再总结一遍研究**
- 用 1-2 句强 take-home + 1-2 句 outlook
- 不要再列样本规模 / 方法细节

### 10. "Future work" 占太多
- Future work 可以提，但不要每个 paragraph 都说"future work could..."
- 1-2 处足够；过多 = reviewer 觉得"现在的研究没做完整"

## 跨方向 illustration

> 元规则的演示，不是范式模板。

### 群体结构 / 系统地理学
- §1 take-home：n 个独立 lineage / specific 群体关系拓扑
- §2-3：与考古、历史、其他物种的 phylogeographic pattern 对话
- §3 implication：保育单元划分 / breeding history 复盘
- §4 limitation：采样空缺 / 古 DNA 缺失

### 驯化 / 人口动态
- §1 take-home：driver 是 climate / human migration / disease / 多重
- §2-3：与考古证据时间线、同时代人类迁徙、家畜驯化中心理论对话
- §3 implication：驯化中心假说的支持 / 驯化时间窗
- §4 limitation：demographic inference 的模型敏感性 / 古 DNA 验证缺失

### 选择信号
- §1 take-home：n 个 high-confidence sweep + 关键候选基因
- §2-3：候选基因的已知功能机制（不要每个基因都写——只挑 top 3-5）
- §3 implication：跨物种 convergent evolution / 育种 marker
- §4 limitation：FST / iHS 等统计量受 demographic confounding / 功能验证缺失

### 渗入 / 杂交
- §1 take-home：渗入比例 / 渗入方向 / 是否 adaptive
- §2-3：与 archaic introgression / domestication-feral hybridization 文献对话
- §3 implication：保育中的 hybrid management / breeding 多样性来源
- §4 limitation：单一参考 panel / 时间分辨率

### 景观基因组学（GEA）
- §1 take-home：n candidate SNPs/genes + 主导环境驱动
- §2-3：candidate gene functional crosswalk + cross-species convergent adaptation
  - 这一段是景观基因组学 Discussion 的**标志段落**：把候选基因和其他物种已发表的 climate adaptation 文献串起来
- §3 implication：保育与气候变化适应 / climate-resilient breeding / genetic offset 风险评估
- §4 limitation：
  - GEA 本身的局限（无法区分 genotype-environment correlation 的真实因果方向）
  - 单方法 vs. 多方法整合的取舍
  - 环境变量代理性（WorldClim 不等于个体经历的真实环境）
  - 候选 SNP 不等于 causal SNP（reviewer 会期待你主动写）

### SV / 泛基因组
- §1 take-home：n SV / n SV-affected genes / SV 与表型连接强度
- §2-3：与短读长漏检的对比 / 与 SNP-only GWAS 的差距
- §3 implication：未来 cattle/sheep 等家畜 pangenome 的应用
- §4 limitation：单系数据集的代表性

### Imputation panel
- §1 take-home：concordance / r² 在某 MAF/coverage 范围下的提升
- §2-3：与现有 panel 在 benchmark 上对比 / 不同物种 panel 的迁移
- §3 implication：低成本基因组学应用（GWAS / 育种 / 保育）
- §4 limitation：reference panel 多样性偏差 / 罕见变异 imputation 仍差

### 多组学整合
- §1 take-home：通过 multi-omic 整合识别的 candidate 数 / 功能解析提升
- §2-3：跟单 omic 研究 / 跟人类同类研究对话
- §3 implication：组学整合的方法学进展
- §4 limitation：tissue-specific data 缺失 / cell-type heterogeneity

## Discussion 的"标志性段落"

（按你确认的方案 B，作为元规则 illustration）

无论方向，**好 Discussion 通常有这 5 类段落**：

1. **Take-home recap**：1 段，强 statement，回应 introduction §4
2. **Mechanism / Functional interpretation**：1-2 段，把 results 放到机制
3. **Cross-context comparison**：1-2 段，与前人 / 跨物种 / 跨方法对话
4. **Implication**：1 段，对应用 / 保育 / 育种 / 临床的潜在价值
5. **Limitations + Future**：1 段，2-4 条具体 limitation + future direction

不是每篇都按 1-2-3-4-5 顺序，但 5 类都要覆盖。**最常见漏掉**：第 5 类（limitation）。

## 自查清单（写完 Discussion 后）

- [ ] **Take-home** 1-2 个，不超过 2 个？
- [ ] 与 Introduction §3 的 gap **闭环**了？
- [ ] 每个 implication 都用了**保守表达**（may / could / suggest / be consistent with）？
- [ ] **Limitations** 有 2-4 条**具体**的（不是"more samples"这种泛话）？
- [ ] 每条 limitation 给了**future direction**？
- [ ] 引用密度合理（每段 3-7 个）？
- [ ] 没有**贬低式**对前人？
- [ ] 没有**重复 Results** 的数字？
- [ ] 没有把 **Discussion 写成 Abstract**？
- [ ] 时态：重述发现过去时 / 解读现在时 / 推测情态动词？
- [ ] AI 痕迹（"highlight the importance of" / "shed light on" / "underscore"）清理过？
- [ ] 篇幅在合理范围（顶刊 1500-3000 词 / 二区可更长）？
